# 2026-05-15 — Dashboard Becomes Captain's Deck

**Status:** Accepted

## Context

The original dashboard was a simple web interface for jobs, metrics, and triggering new runs. As the Fleet concept emerged, the dashboard needed to become more than a utility table.

## Decision

Evolve the dashboard into **Captain's Deck**, the command center for NayzFreedom Fleet.

Phase direction:

1. Build Fleet shell.
2. Upgrade The Aurora.
3. Add crew portraits and character sheets.
4. Make the dashboard mobile-friendly.
5. Prepare for VPS and command integrations.
6. Start The Freedom + Nami after memory/privacy structure is clearer.

## Consequences

- The dashboard now uses ship-based navigation.
- `/aurora` is the flagship operational page.
- Crew and character identity are first-class UI concepts.
- Mobile and production readiness must be considered before deeper personal systems.

## Follow-ups

- Implement Nami and The Freedom after logging/memory strategy is in place.
